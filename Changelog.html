NSBEdit Version 2026.05.29.10 (Fix: RTF formatting lost on session restore)

Fix: RTF formatting (bold, italic, underline, etc.) lost on session restore (29.05.2026, 10:09):

  • Fix: RTF formatting lost on session restore: After a restart, unsaved RTF tabs had all character formatting stripped — bold text became plain, italic became plain, and so on. Root cause: the session-restore colour fixup used CFM_EFFECTS as the dwMask in EM_SETCHARFORMAT. Since CFM_EFFECTS covers all effect bits (bold, italic, underline, strikethrough…), and dwEffects only had CFE_AUTOCOLOR set, RichEdit zeroed out every other effect on every character. Fixed by changing the mask to CFE_AUTOCOLOR only, so just the autocolor flag is written and all per-character bold / italic / underline / colour runs from the streamed-in RTF are preserved intact.

NSBEdit Version 2026.05.26.10 (Scrollbar tab-switch fix)

Custom scrollbar fade-state fix — bars no longer disappear after switching tabs (26.05.2026, 10:53):

  • Fix: custom scrollbars vanish after switching tabs: Switching away from a tab and back caused the custom scrollbar windows to disappear and stay hidden for the lifetime of the session. Root cause: Ne_SyncScrollbarVisibility was calling ShowWindow(hBar, SW_HIDE) directly on the bar HWND when hiding an inactive tab, bypassing the scrollbar library’s internal fadeState field. On re-activation, msb_reposition calls Msb_UpdateVisibility, which only issues ShowWindow(…SW_SHOWNOACTIVATE) when fadeState == FADE_INVISIBLE. Ne_SyncScrollbarVisibility now calls msb_hide() instead of ShowWindow directly, so the next msb_reposition / msb_sync call correctly re-shows the bar. Affects all four bar handles (RichEdit V/H and Scintilla V/H).
